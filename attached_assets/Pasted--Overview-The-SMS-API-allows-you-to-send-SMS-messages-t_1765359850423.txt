
Overview
The SMS API allows you to send SMS messages through our bulk and shortcode messaging gateway. This documentation provides everything you need to integrate SMS functionality into your application.
 Note: You need an API Key to use this API. If you have already signed up, you can get your API Key from your account.
API Endpoint
POSThttps://app.mobitechtechnologies.com//sms/sendsmsBulk SMS
Authentication
All API requests require authentication using your API key in the request header.
Header Parameter
Header
Type
Description
Required
h_api_key
String
Your API key (64 characters)
Yes
Content-Type
String
Must be application/json
Yes

Example Header
cURLPHP

--header 'h_api_key: ba5d2c53db7329b470c329c17765aea9e0ae6b34018f3a3f' \
--header 'Content-Type: application/json'
Send Single SMS
Send a message to a mobile subscriber using a Sender Name (alphanumeric sender ID).
POSThttps://app.mobitechtechnologies.com//sms/sendsmsBulk SMS
Request Parameters
Field
Type
Description
Required
mobile
String
The customer mobile number (e.g., 722xxx, 0722xxx, +254722xxx)
Yes
sender_name
String
The origination alphanumeric or numeric code (e.g., "MOBI-TECH")
Yes
message
String
The message to send (max 920 characters or 6 SMS units)
Yes
response_type
String
[Optional] Either "json" or "plain", defaults to "json"
No
service_id
Integer
Service identifier (always 0 for bulk messaging)
Yes
link_id
String
[Optional] For shortcode messages only
No

Code Examples
cURLPHP

curl --location --request POST 'https://app.mobitechtechnologies.com//sms/sendsms' \
--header 'h_api_key: ba5d2c53db7329b470c329c17765aea9e0ae6b34018f3a3f' \
--header 'Content-Type: application/json' \
--data-raw '{
    "mobile": "+254710986455",
    "response_type": "json",
    "sender_name": "MOBI-TECH",
    "service_id": 0,
    "message": "This is a message.\n\nRegards\nMobitech Technologies Ltd"
}'

Send Multiple SMS
Send multiple SMS messages to different recipients in a single API call.
POSThttps://app.mobitechtechnologies.com//sms/sendmultipleBulk SMS
Request Parameters
Field
Type
Description
Required
serviceId
Integer
The Service Id (0 for bulk SMS)
Yes
shortcode
String
Sender Name for whitelabeling
Yes
messages
Array
List of message objects
Yes
messages[].mobile
String
Recipient phone number
Yes
messages[].message
String
Message content
Yes
messages[].client_ref
String/Integer
Reference for delivery reports
No

Code Example
cURLPHP

curl --location --request POST 'https://app.mobitechtechnologies.com//sms/sendmultiple' \
--header 'Content-Type: application/json' \
--header 'h_api_key: ba5d2c53db7329b470c329c17765aea9e0ae6b34018f3a3f' \
--data-raw '{
    "serviceId": "0",
    "shortcode": "MOBI-TECH",
    "messages": [
        {
            "mobile": "+254710986455",
            "message": "This is a first message",
            "client_ref": 6481
        },
        {
            "mobile": "+254712244244",
            "message": "This is a second message",
            "client_ref": 6092
        }
    ]
}'

Delivery Reports
Receive delivery reports from Mobile Network Operators in real-time.
POST/client.endpoint/dlrShortcode
 Note: This endpoint should be configured on your server to receive delivery reports.
Request Parameters
Field
Type
Description
Required
messageId
Integer
The messageId returned with every successful request
Yes
dlrTime
DateTime
DateTime for the time of delivery (YYYY-MM-DD HH:MM:SS)
Yes
dlrStatus
Integer
Delivery status value as integer
Yes
dlrDesc
String
Description of the delivery report status
Yes
tat
Integer
Time taken for system to get report (in seconds)
No
network
Integer
Numeric indicator of the Network
No
destaddr
String
The mobile number the message was sent to
Yes
sourceaddr
String
The SenderId which originated the message
Yes
origin
String
The network name (e.g., "Safaricom")
No

Example Delivery Report
cURLPHP

curl --location --request POST '/client.endpoint/dlr' \
--header 'Content-Type: application/json' \
--data-raw '{

    "dlrTime": "2022-02-22 11:46:42",
    "dlrStatus": "1",
    "dlrDesc": "DeliveredToTerminal",
    "tat": 10005,
    "network": "1",
    "destaddr": "254767209917",
    "sourceaddr": "118",
    "origin": "Safaricom"
